# 项目介绍
该项目提示词适用于`trae`或`cursor`或`kiro`这类AI编辑器，适用于多轮对话的形式去推进阶段任务<br>
目前来看，`glm4.7`的能力还是不错的，同时因为是免费，所以目前推荐`trae cn`的方式去调用审计，时不时需要复查而已<br>
`claude code`的skill正在编写中，目前还在测试，想到了一种还不错的方式，可以期待一下（已更新）


# 代码审计思路
1.先通过阶段一，对架构、权限控制、接口路由等内容进行梳理，在利用第二次、第三次的复查，确保阶段一内容无误
2.不断的往下推进，且根据需求对特定阶段的任务进行复查
3.整合报告，并进行漏洞之间的组合利用

# 使用指南（文档版）
创建项目后，将文件放到项目根目录中，然后跟ai进行对话<br>
推进任务
```
阅读代码审计.md文件，开始推进阶段一的任务
```
复查阶段一(简略版):
```
阅读代码审计.md文件，复查阶段一的结果，检查是否有接口遗漏需要补充的，或者是接口路径不全需要补充的
```
复查阶段一：
```
仔细审查代码审计.md文件中阶段一的所有检查结果，重点关注以下方面：
1. 接口地址清单是否完整无遗漏，每个接口地址是否包含完整的URL路径、请求方法(GET/POST等)和功能描述
2. 架构设计文档是否全面，包括但不限于：
   - 系统组件图及交互关系
   - 数据流向示意图
   - 关键模块设计说明
3. 权限控制机制是否完整覆盖所有接口，确保：
   - 每个接口都有明确的权限要求说明
   - 权限验证逻辑描述清晰
   - 特殊权限场景(如管理员权限、跨部门访问等)都有相应说明

如有需要，补充更新到代码审计.md中
```
复查其他阶段
```
阅读代码审计.md或各阶段报告(如存在)，复查其中的漏洞，判断是否是误报或者夸大危害
```
报告整合（适用于 单独输出报告文件版）
```
阅读代码审计.md文件和各阶段的输出报告，然后开始推进阶段xx的任务
```

# 使用指南（skills版）
直接让他加载技能，然后完成所有阶段的任务即可
```
/code-audit 加载代码审计技能，完成所有阶段的任务
```

# 提示词区别
在使用trae的时候发现，solo模型也支持并行（用trae的原因是因为glm4.7免费，且审计能力来说，有耐心多复查几遍，和claude审计出来的结果差不多），所以单独改了一下，把除了第一阶段的报告都单独输出，这样就可以在每个任务都不会相互影响的情况下，读取第一阶段的内容去针对性的扫描
